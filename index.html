オープンソースライセンスは、貢献者とユーザーを保護します。ビジネスや熟練した開発者は、この保護がないプロジェクトに手を出しません。

{ 以下のうち、あなたの状況にもっとも適したものはどれですか？ }

  • コミュニティで作業する必要があります。

    貢献するまたは依存するコミュニティが好むライセンスを使用してください。あなたのプロジェクトはうまく適合するでしょう。

    ライセンスがない依存関係がある場合は、そのメンテナにライセンスを追加するように依頼してください。

  • シンプルで許容的なものが欲しいです。

    MITライセンスは簡潔で明確です。プロジェクトに対してほとんど何でもできるようになります。たとえば、クローズドソースのバージョンを作成して配布できます。

    {% include using-sentence.html license-id="mit" %}

  • 改良を共有することが重要です。

    GNU GPLv3は、プロジェクトに対してほとんど何でもできるようになりますが、クローズドソースのバージョンを配布することはできません。

    {% include using-sentence.html license-id="gpl-3.0" %}

{ もし私にとってこれらのどれもが適切でなかった場合は？ }

  • 私のプロジェクトはソフトウェアではありません。

    それに対応するライセンスがあります。

  • もっと選択肢が欲しいです。

    さらに多くのライセンスがあります。

  • ライセンスを選択したくありません。
